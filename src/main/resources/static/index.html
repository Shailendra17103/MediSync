API Endpoints for Postman

Use these examples in your Postman requests

GET /api/patients

Retrieves a list of all patients

POST /api/patients

Creates a new patient

Example payload
{ "firstName": "Ava", "lastName": "Sharma", "email": "ava@example.com", "dateOfBirth": "1995-04-12", "gender": "FEMALE", "phoneNumber": "+12025550123" }
GET /api/patients/{id}

Retrieves a specific patient by ID

PUT /api/patients/{id}

Updates an existing patient

Example payload
{ "firstName": "Ava", "lastName": "Kapoor", "email": "ava@example.com", "dateOfBirth": "1995-04-12", "gender": "FEMALE", "phoneNumber": "+12025550123" }
DELETE /api/patients/{id}

Deletes a patient

Validation Rules

These rules are enforced when creating or updating patients

Field Validation Error Message
firstName Required, max 100 chars "First name is required"
lastName Required, max 100 chars "Last name is required"
email Required, valid format, unique "Invalid email format"
dateOfBirth Must be in the past "Date of birth must be in the past"
gender Required (MALE, FEMALE, OTHER, PREFER_NOT_TO_SAY) "Gender is required"
phoneNumber Optional, E.164 format "Phone number must be E.164 format"
